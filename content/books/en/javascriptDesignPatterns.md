I have to confess that I skipped some chapters because they did not cover something I could use in the immediate future. In it, Addy Osmani talks about the types of design patterns and gives examples of them. It is a very complete book that covers everything from the most common patterns in Javascript to the most complex. It is a book we can say is essential; it dedicates most of the book to design patterns in "vanilla" Javascript, but in the final part it tells us how jQuery applies those patterns, which is very interesting, especially if you have used it. Nowadays jQuery is falling out of use, but its code is still totally valid and it remains present in many CMSs like WordPress, for example. I found the section on modular patterns like AMD (Asynchronous Module Definition) especially interesting. For me, it is a "must have," though more as a reference than as a technical read from which to extract 100% of what it says. This book should be read calmly; it is not a novel.
